# Welcome to Çukurova University Cafeteria

A SaaS web application that displays daily cafeteria menus for Çukurova University. The app scrapes data from the university's official website, transforms it into structured JSON, and presents it with powerful features for students and staff.

**Live Demo:** [https://cukurova.vercel.app](https://cukurova.vercel.app)

## Technical architecture

The project follows an **Extract-Transform-Load (ETL)** data pipeline:

<Steps>
  <Step title="Data extraction (Scraping)">
    A Node.js script (`scripts/scrape-menu.ts`) uses `cheerio` to parse the university's ASP-based website
  </Step>
  
  <Step title="Data transformation (Parsing)">
    Raw HTML is converted from `Windows-1254` to `UTF-8` using `iconv-lite`, then parsed to extract day, meal, and calorie information
  </Step>
  
  <Step title="Data storage">
    Processed data is saved as timestamped JSON files in `public/data/<YYYY-MM>/` directories. Each month folder keeps a maximum of 5 files
  </Step>
  
  <Step title="Presentation">
    The Next.js application reads these static JSON files server-side using Server Components and delivers them to the client
  </Step>
</Steps>

## Technology stack

Built with modern, production-ready technologies:

**Frontend**
- **Framework:** Next.js 16 with App Router and Server Components
- **UI Library:** React 19
- **Language:** TypeScript
- **Styling:** Tailwind CSS 4
- **Components:** shadcn/ui + Radix UI
- **Icons:** Lucide React
- **Animations:** Framer Motion

**Backend**
- **Database:** NeonDB (Serverless PostgreSQL)
- **ORM:** Drizzle ORM + Drizzle Kit
- **Authentication:** NextAuth.js v5 (Auth.js) + Google OAuth
- **Email:** nodemailer with Google SMTP

**Features**
- **Scraping:** cheerio, iconv-lite
- **Image Generation:** html-to-image
- **Calendar:** react-day-picker, date-fns
- **Forms:** react-hook-form, zod
- **Notifications:** sonner
- **PWA:** @ducanh2912/next-pwa
- **Analytics:** Vercel Analytics, Google Analytics

**Package Manager:** pnpm

## Automated updates

<Note>
The application uses GitHub Actions to automatically scrape menu data every day at 07:00 (UTC+3) and send email notifications for favorite meals.
</Note>

**Workflow features:**
- Triggers daily at UTC 04:00 (Turkey time 07:00) or manually via `workflow_dispatch`
- Runs `npm run scrape` to fetch the latest menu data
- Commits and pushes changes to `public/data/` if new data is found
- Executes `send-favorite-notifications.ts` to match today's meals with user favorites
- Sends email notifications via Google SMTP when matches are found

<Warning>
GitHub Secrets required: `DATABASE_URL`, `SMTP_USER`, `SMTP_PASS`
</Warning>
