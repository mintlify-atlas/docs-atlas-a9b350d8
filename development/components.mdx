## Component architecture

The app uses a component-based architecture with:
- **shadcn/ui** for base UI primitives
- **Radix UI** for accessible components
- **Framer Motion** for animations
- **Client/Server Component** separation for performance

## Core components

### MenuCard

Displays a daily menu with meals, calories, reactions, and AI prompts.

**Location**: `components/menu-card.tsx`

**Props**:

```typescript
interface MenuCardProps {
  day: MenuDay  // Menu data for a specific day
  onMealClick: (mealId: string, mealName: string, mealCalories: number) => void
}
```

**Usage**:

```tsx
import { MenuCard } from "@/components/menu-card"

<MenuCard 
  day={menuDay} 
  onMealClick={(id, name, cals) => openMealModal(id)} 
/>
```

**Features**:
- Like/dislike buttons
- Favorite toggle per meal
- "I ate this" button for calorie tracking
- AI prompt generation (ChatGPT, Claude, Grok, Perplexity)
- Menu sharing as image
- Responsive layout

<Accordion title="MenuCard implementation details">
  The component:
  - Fetches daily log on mount via `getDailyLog(day.date)`
  - Shows `CircleCheck` icon for consumed meals
  - Prevents future date meal logging
  - Prompts for calorie goal on first use
  - Uses optimistic updates for instant feedback
  - Displays total calories with color indicator (green < 800, amber < 1100, red ≥ 1100)
</Accordion>

### LikeDislikeButtons

Anonymous reaction buttons with optimistic updates and rate limiting.

**Location**: `components/like-dislike-buttons.tsx`

**Props**:

```typescript
interface LikeDislikeButtonsProps {
  menuDate: string  // "2026-02-19" format
}
```

**Usage**:

```tsx
<LikeDislikeButtons menuDate="2026-02-19" />
```

**Features**:
- IP-based rate limiting (prevents spam)
- Optimistic UI updates (instant feedback)
- localStorage persistence (remembers user choice)
- Animated counters (only show count > 0)
- Automatic rollback on error

<Accordion title="Reaction state machine">
  State transitions:
  - `null → like`: Add like
  - `like → null`: Remove like
  - `like → dislike`: Remove like, add dislike
  - `dislike → like`: Remove dislike, add like
  
  LocalStorage key: `reaction_{menuDate}`
</Accordion>

### MealDetailModal

Shows meal ingredients and image scraped in real-time.

**Location**: `components/meal-detail-modal.tsx`

**Props**:

```typescript
interface MealDetailModalProps {
  mealId: string | null
  mealName: string
  onClose: () => void
}
```

**Usage**:

```tsx
<MealDetailModal 
  mealId="157" 
  mealName="Ekşili Köfte"
  onClose={() => setMealId(null)} 
/>
```

**Behavior**:
- Fetches from `/api/meal/[id]` on open
- Displays loading skeleton
- Shows ingredients table (name, amount, unit)
- Meal image (if available)
- Desktop: Dialog, Mobile: Drawer

### DatePicker

Calendar date range picker with mobile bottom sheet.

**Location**: `components/date-picker.tsx`

**Props**:

```typescript
interface DatePickerProps {
  availableDates: string[]  // ["2026-02-19", "2026-02-20"]
  currentDate: string
  onDateSelect: (date: string) => void
}
```

**Usage**:

```tsx
<DatePicker 
  availableDates={dates}
  currentDate={selectedDate}
  onDateSelect={setSelectedDate}
/>
```

**Features**:
- Highlights available dates
- Disables unavailable dates
- Mobile: Bottom sheet drawer
- Desktop: Popover dialog
- `react-day-picker` integration

### AuthButton

Google OAuth login with profile dropdown.

**Location**: `components/auth-button.tsx`

**States**:
- **Not logged in**: "Giriş Yap" button
- **Logged in**: User avatar + dropdown menu

**Dropdown menu**:
- Profile info (name, email)
- Navigate to Favorites
- Navigate to Calorie Tracking
- Sign out

**Usage**:

```tsx
import { AuthButton } from "@/components/auth-button"

<AuthButton />
```

### ThemeToggle

Switch between light, dark, and system theme.

**Location**: `components/theme-toggle.tsx`

**Usage**:

```tsx
import { ThemeToggle } from "@/components/theme-toggle"

<ThemeToggle />
```

**Themes**:
- Light
- Dark
- System (follows OS preference)

Uses `next-themes` for persistence.

## Animation patterns

### Framer Motion scale

```tsx
import { motion } from "framer-motion"

<motion.button whileTap={{ scale: 0.95 }}>
  Click me
</motion.button>
```

### AnimatePresence for enter/exit

```tsx
import { AnimatePresence, motion } from "framer-motion"

<AnimatePresence mode="wait">
  {show && (
    <motion.div
      initial={{ opacity: 0, y: 10 }}
      animate={{ opacity: 1, y: 0 }}
      exit={{ opacity: 0, y: -10 }}
    >
      Content
    </motion.div>
  )}
</AnimatePresence>
```

## Responsive design

Use Tailwind breakpoints:

```tsx
<div className="hidden md:block">Desktop only</div>
<div className="block md:hidden">Mobile only</div>
```

Or use the `useMobile` hook:

```tsx
import { useIsMobile } from "@/hooks/use-mobile"

const isMobile = useIsMobile()

return isMobile ? <MobileView /> : <DesktopView />
```
