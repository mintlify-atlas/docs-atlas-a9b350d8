The scraping system automatically fetches menu data from Çukurova University's legacy ASP website, transforms it into structured JSON, and stores it for the Next.js application to consume.

## Scraping workflow

### Automated execution

The scraper runs automatically via GitHub Actions:

```yaml
# .github/workflows/menu-update.yml
schedule:
  - cron: '0 4 * * *'  # Every day at 04:00 UTC (07:00 UTC+3)
```

After scraping:
1. New JSON files are committed to the repository
2. Vercel detects the commit and auto-deploys
3. Favorite notification script runs to email users

### Manual execution

You can also run the scraper manually:

```bash
pnpm scrape
```

This executes `scripts/scrape-menu.ts` using tsx (TypeScript executor).

## Scraper implementation

The scraper is implemented in `scripts/scrape-menu.ts` and uses the core logic from `lib/scraper.ts`.

### Step 1: Fetch HTML with proper encoding

The university website uses **Windows-1254** encoding (Turkish characters), which requires special handling:

```typescript
import iconv from 'iconv-lite';

const response = await fetch('https://yemekhane.cu.edu.tr/default.asp');
const buffer = await response.arrayBuffer();
const html = iconv.decode(Buffer.from(buffer), 'windows-1254');
```

<Note>
Without `iconv-lite`, Turkish characters (ğ, ü, ş, ı, ö, ç) would display incorrectly. The library converts Windows-1254 to UTF-8.
</Note>

### Step 2: Parse HTML with Cheerio

Cheerio provides a jQuery-like API for server-side HTML parsing:

```typescript
import * as cheerio from 'cheerio';

const $ = cheerio.load(html);
const dayCards = $('.col-md-2, .col-md-3');
```

Each day's menu is contained in a Bootstrap grid column (`col-md-2` or `col-md-3`).

### Step 3: Extract day information

For each day card, extract:

<CodeGroup>

```typescript Date and YMK
// YMK (Yemekhane ID) from link: ?ymk=123
const dateLink = $card.find('FONT[size="6"] a').attr('href');
const ymkMatch = dateLink?.match(/ymk=(\d+)/);
const ymk = ymkMatch ? parseInt(ymkMatch[1]) : null;

// Date text: "03.11.2025 Pazartesi"
const $font = $card.find('FONT[size="6"]');
$font.find('br').replaceWith(' ');
let dateText = $font.text().replace(/\s+/g, ' ').trim();
const dateMatch = dateText.match(/(\d{1,2})\.(\d{2})\.(\d{4})/);

const [, day, month, year] = dateMatch;
const date = `${year}-${month}-${day.padStart(2, '0')}`; // "2025-11-03"
```

```typescript Day name
// Extract day name (e.g., "Pazartesi")
let dayName = dateText.split(/\s+/)[1] || '';
if (!dayName) {
  dayName = $card.find('FONT[size="5"]').text().split(',')[0].trim() || '';
}
```

```typescript Data availability check
// Check if menu data exists for this day
const cardText = $card.text();
const hasData = !cardText.includes('Yemekhane yemek bilgileri girilmemiştir');
```

</CodeGroup>

### Step 4: Extract meals

If `hasData` is true, parse individual meals:

```typescript
const meals: Meal[] = [];

$card.find('ul li a').each((i, el) => {
  const $link = $(el);
  const href = $link.attr('href') || '';
  
  // Extract meal ID: yemek-goster.asp?id=157 → "157"
  const idMatch = href.match(/id=(\d+)/);
  if (!idMatch) return;
  const id = idMatch[1];
  
  // Parse meal name and calories from link HTML
  const linkHtml = $link.html() || '';
  const parts = linkHtml.split('<br>');
  const name = parts[0]?.trim() || '';
  
  // "294 Kalori" → 294
  const caloriesText = parts[1] || '';
  const caloriesMatch = caloriesText.match(/(\d+)/);
  const calories = caloriesMatch ? parseInt(caloriesMatch[1]) : 0;
  
  meals.push({ id, name, calories, category: 'ana_yemek' });
});
```

### Step 5: Extract total calories

```typescript
let totalCalories = 0;

const calorieSpans = $card.find('span[style*="color: rgb(204, 0, 0)"]');
calorieSpans.each((i, el) => {
  const text = $(el).text();
  // "Toplam 797 Kalori" or "Ana Yemekli : 891 Kalori"
  const match = text.match(/(\d+)/);
  if (match) {
    const cal = parseInt(match[1]);
    // Prefer "Ana Yemekli" value if available
    if (text.includes('Ana Yemekli') || i === 0) {
      totalCalories = cal;
    }
  }
});
```

### Step 6: Construct DayMenu object

```typescript
days.push({
  ymk: ymk!,
  date,
  dayName,
  hasData,
  meals,
  totalCalories,
});
```

### Step 7: Build MenuData object

```typescript
const today = new Date();
const year = today.getFullYear();
const monthNum = String(today.getMonth() + 1).padStart(2, '0');
const dayNum = String(today.getDate()).padStart(2, '0');

const scrapeDate = `${year}-${monthNum}-${dayNum}`;
const month = `${year}-${monthNum}`;

const menuData: MenuData = {
  month,
  lastUpdated: new Date().toISOString(),
  scrapeDate,
  totalDays: days.length,
  days: days.sort((a, b) => a.date.localeCompare(b.date)),
};
```

## File storage strategy

The script saves JSON files to `public/data/<YYYY-MM>/` directories:

### Directory structure

```
public/data/
├── 2026-01/
│   ├── menu-20260131.json
│   └── ...
└── 2026-02/
    ├── menu-20260227.json
    ├── menu-20260228.json
    ├── menu-20260301.json
    ├── menu-20260302.json
    └── menu-20260303.json
```

### File naming convention

```typescript
// Format: menu-YYYYMMDD.json
const scrapeDateCompact = menuData.scrapeDate.replace(/-/g, '');
const filename = `menu-${scrapeDateCompact}.json`;
// Example: menu-20260302.json
```

### File cleanup logic

To prevent unlimited file accumulation, the scraper keeps only the **5 most recent files** per month:

```typescript
const files = fs.readdirSync(monthDir)
  .filter(file => file.endsWith('.json') && file.startsWith('menu-'))
  .map(file => ({
    name: file,
    path: path.join(monthDir, file)
  }))
  .sort((a, b) => b.name.localeCompare(a.name)); // Newest first

if (files.length > 5) {
  const filesToDelete = files.slice(5);
  filesToDelete.forEach(file => {
    fs.unlinkSync(file.path);
    console.log(`🗑️ Deleted: ${file.name}`);
  });
}
```

## Limitations and considerations

<Warning>
The scraper depends on the HTML structure of the university website. Any changes to the website's markup could break the parser.
</Warning>

### Known issues

- **Legacy HTML**: Website uses deprecated `<FONT>` tags and inline styles
- **Encoding**: Windows-1254 encoding requires special handling
- **Dynamic content**: Website doesn't use modern JavaScript frameworks (easier to scrape)
- **Rate limiting**: No rate limiting on university website (be respectful)

### Best practices

1. **Run once daily**: GitHub Actions scheduled for 07:00 UTC+3
2. **Handle encoding properly**: Always use `iconv-lite` for Windows-1254
3. **Validate data**: Check for `hasData` before parsing meals
4. **Clean up old files**: Maintain only 5 files per month
5. **Log errors**: Use structured logging for debugging
